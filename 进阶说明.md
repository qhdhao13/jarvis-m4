# JARVIS-M4 进阶环节说明

基础骨架已就绪（耳朵 → 大脑 → 嘴巴），可按以下方向逐步增强。

---

## 常见问题：能说多久？能记住吗？理解了吗？

- **能接受多长时间的说话？**  
  单次录音最长由 `config.yaml` 里 **`ears.timeout_seconds`** 决定，默认 **15 秒**。说完一句话后，若持续静音约 0.5 秒（`silence_duration_ms`），会自动结束这句并送去识别。想说更长（例如一段话 20～30 秒），把 `ears.timeout_seconds` 改为 `20` 或 `30` 即可。

- **能记住吗？**  
  能。同一轮会话里会保留**对话历史**；对话多了会做**摘要压缩**（见下方「记忆 / RAG」），所以会记住前面说过的大意。最多保留约 10 条消息，每 8 条会生成一条摘要，相当于能记住多轮对话的要点。  
  **注意**：关掉贾维斯再重新打开，会话会清空（当前未做持久化到磁盘）。

- **理解了吗？**  
  能。每次回复都会把「系统提示词 + 对话历史（含摘要）」发给大脑（LLM），所以贾维斯会结合你**之前说过的话**来理解「你」「刚才」「之前说的那个」等指代，理解是**连贯的**。

## 一、唤醒词（已实现可选集成）

- **目标**：不用按 Enter，说出唤醒词即开始录音。
- **现状**：`config.yaml` 中 `wake_word.enabled: true` 时使用 openWakeWord 监听；为 `false` 时仍为「按 Enter 后录音」。
- **首次使用**：安装依赖后需先下载预训练模型（需联网一次）。若遇到 **SSL 错误**（如 `SSLError`、`EOF occurred in violation of protocol`），说明本机 Python 的 SSL 与 GitHub 不兼容，请用项目提供的脚本（用系统 curl 下载，绕过 Python SSL）：
  ```bash
  ./scripts/download_openwakeword_models.sh
  ```
  网络正常时也可用官方方式：
  ```bash
  python3 -c "import openwakeword; openwakeword.utils.download_models()"
  ```
  若未下载就启用唤醒词，程序会自动退回「按 Enter 后录音」并提示下载。
- **预训练模型**：库自带英文唤醒词如 **Hey Jarvis**、alexa 等；若要「贾维斯」中文唤醒，需用官方文档自训练并填写 `wake_word.model_path`。
- **依赖**：`pip3 install openwakeword`（已加入可选依赖说明）。
- **国内镜像说明**：阿里云、清华等镜像站主要提供 **git 仓库克隆**，不代理 GitHub 的 release 文件直链，所以 openWakeWord 的模型没有阿里/清华直链。脚本会依次尝试 **ghproxy.net / ghproxy.com / mirror.ghproxy.com** 等 GitHub 加速代理；也可设置本机代理 `export HTTPS_PROXY=...` 后直接连 GitHub。

## 二、记忆 / RAG

- **目标**：让贾维斯记住多轮对话要点，或基于本地文档回答问题。
- **可选做法**：
  - **对话摘要**：每 N 轮用 LLM 生成简短摘要，追加到 system 或单独上下文。
  - **向量库**：用 LangChain + 本地向量库（如 Chroma、FAISS），把文档切片入库，回答前先检索再交给 LLM。
- **建议**：先做对话摘要（改 `brain` 与 `main` 的 history 处理），再考虑文档 RAG。

## 三、控制电脑

- **目标**：语音控制本机（打开应用、调节音量、锁屏等）。
- **现状**：`brain` 已有 `exec_shell` 工具，可执行任意命令。
- **进阶**：
  - 在 system 提示词中明确可执行的安全命令列表（如 `open -a Safari`、`osascript` 调音量）。
  - 增加「意图 → 命令」映射，避免直接暴露 shell；或限制为白名单命令。

---

## 建议顺序

1. **唤醒词**：先打通「说完唤醒词 → 自动录音」的体验（当前已支持可选开启）。
2. **控制电脑**：在现有 `exec_shell` 上做白名单与提示词约束，即可用语音控制常用操作。
3. **记忆 / RAG**：在对话稳定后再加摘要或文档检索，避免一次性改太多。

如需调整顺序或先做其中某一项，可直接说明。

## 四、双击启动（启动贾维斯.command）

- **目标**：在 Finder 中双击即可启动贾维斯，并同时打开状态动画图标小窗。
- **行为**：
  1. 双击 `启动贾维斯.command` 后，会先在后端启动主程序（`python3 main.py`），约 1 秒后启动状态图标（`python3 jarvis_icon.py`）。
  2. 状态图标窗口在前台显示（待机/听/思考/说 的变色圆窗）；主程序在后台运行。
  3. **关闭状态图标窗口即会同时结束主程序**：脚本在图标进程退出后会向主进程发送结束信号，因此关掉小窗即可完全退出贾维斯。
  4. 若从终端用 Ctrl+C 先结束主程序，图标窗口可能仍会保留直到你关掉它；此时再关窗口即可。
- **建议**：需要看终端日志时，用 `./run.sh` 或 `python3 main.py`；仅需语音对话与状态窗时，用双击 `启动贾维斯.command` 即可。

## 五、状态动画图标

- **目标**：显示随状态变化的动画图标（待机/听/思考/说）。
- **用法**：
  1. 确保 `config.yaml` 中已设置 `icon_state_file: ".jarvis_state"`（已默认写入）。
  2. 先启动贾维斯（`./run.sh` 或 `python3 main.py`），再在**另一个终端**运行：`python3 jarvis_icon.py`。
  3. 或直接双击 `启动贾维斯.command`，会同时启动主程序与图标（见第四节）。
  4. 会弹出一个小圆窗，随状态变色与脉动（IDLE 灰、LISTENING 蓝、PROCESSING 黄、SPEAKING 绿）。
- **依赖**：仅需 Python 自带的 tkinter。

## 六、指挥 OpenClaw

- **目标**：贾维斯通过本机已安装的 OpenClaw，发送指令给 OpenClaw 智能体（截屏、相机、UI 自动化等）。
- **现状**：已增加工具 `openclaw_send(message)`，通过 OpenClaw 深度链接 `openclaw://agent?message=...` 发送自然语言指令。你说「让 OpenClaw 截屏」「让 OpenClaw 打开灯」等，贾维斯会转发给 OpenClaw。
- **前提**：本机已安装并运行 **OpenClaw.app**（菜单栏），且 Gateway 已连接；首次可能需在 OpenClaw 内确认执行。

### 执行结果语音回传（可选）

- **目标**：OpenClaw 执行完后，贾维斯用语音把结果读给你听，而不是只说「已发送，请在界面查看」。
- **做法**：
  1. 在 OpenClaw 的 **Gateway 配置**中开启 Responses HTTP 端点，并设置认证（Token 或 Password）。例如在 Gateway 配置文件中加入：
     ```json5
     { gateway: { http: { endpoints: { responses: { enabled: true } } }, auth: { mode: "token", token: "你的密钥" } }
     ```
  2. 在 JARVIS 的 `config.yaml` 中把 `openclaw.use_http` 改为 `true`，并设置 `openclaw.gateway_url`（默认 `http://127.0.0.1:18789`）。**Token 建议用环境变量**，不要写在 config 里：`export OPENCLAW_GATEWAY_TOKEN=你的密钥`；若 Gateway 使用 password 模式，则用 `OPENCLAW_GATEWAY_PASSWORD`，并在 Gateway 侧配置一致。
  3. 重启贾维斯。之后当你说「让 OpenClaw 做 xxx」时，贾维斯会通过 Gateway 的 `POST /v1/responses` 发送指令并等待执行结果，再把返回的文本用语音播报；若未配置或请求失败，会自动退回「仅发深度链接 + 语音说请在界面查看」。
- **参考**：OpenClaw 文档 [OpenResponses HTTP API](https://docs.openclaw.ai/gateway/openresponses-http-api)、[Gateway 运维](https://docs.openclaw.ai/gateway)。

## 七、其它说明

- **run.sh 与 config 配合**：`run.sh` 会读取 `config.yaml` 中的 `llm.backend`。若为 `kimi` 且已设置环境变量 `KIMI_API_KEY`，则不会检查 Ollama 是否启动；若为 `ollama`（默认），则会检查 Ollama 服务是否可达，未启动时会提示。
- **可选依赖**：见项目根目录 `requirements.txt` 顶部注释（唤醒词 openwakeword、模型下载脚本等）。
